# 5 分钟演示视频脚本

## 0:00-0:30 问题提出

大家好，本作品是“AI 生成图像取证研判与警务证据链系统”，聚焦社交平台传播扰动后的 AIGC 图像辅助核查。

GPT-image-2 等生成模型可以快速生成高度拟真的公共安全场景图片。图片一旦进入社交平台传播，可能经历 JPEG 压缩、截图转存、裁剪、水印覆盖和二次上传，C2PA 元数据或平台标识可能被剥离。因此，单靠元数据溯源并不可靠，需要在元数据失效时提供可复核的视觉检测线索。

## 0:30-1:30 技术路线

我们的技术路线不是简单套一个大模型接口，而是建设本地监督训练闭环。

第一，建设外部数据训练池。当前生成模型归因训练池共有 4691 张图片，覆盖真实照片、GPT-image-2、Flux、SDXL、Midjourney、DALL-E、Seedream、Nano Banana 等类别，内置展示样例不进入训练集。

第二，抽取多组本地特征，包括视觉语义、频域纹理、JPEG 压缩痕迹、8x8 块边界差异、文字覆盖和水印代理特征。

第三，训练本地视觉检测/归因头。当前 active 模型为 e19d4bb3 开头版本，使用 ExtraTrees 分类器和类别原型兜底，低置信时输出 unknown。提交口径上，我们把主线收束为低误报真实/生成初筛和 GPT-image-2 疑似来源线索。

第四，做传播扰动增强。训练 split 生成 1200 条临时扰动特征，验证集保持 clean holdout，防止验证污染。

## 1:30-3:30 系统演示

打开系统首页。标题是“AI 生成图像取证研判中心”。顶部可以看到训练池规模、active 归因头、扰动增强数量和传播扰动复测对比。

选择一个案例，先上传待核验图片。系统保存原始文件、MIME、大小、宽高、sha256 和上传时间。

点击启动图像取证研判。系统读取 active 本地归因头，对图片输出疑似生成来源线索、置信度、候选分布、传播扰动迹象、Top 特征贡献和不确定项。这里所有 GPT-image-2 相关表述都只作为疑似来源线索，不作为确定结论。

切换到证据链页面。可以看到图像证据、来源证据、传播证据和模型分析条目，每个条目带有 hash、来源、置信度或审计 ID，方便人工复核。

切换到处置页面。系统根据证据链给出核查原图、保全传播链路、联系平台协查、公开澄清和人工复核建议。

最后打开报告页面。报告会写入模型版本、图片 hash、审计 ID、知识依据和人工复核声明，可以直接作为半成品文书草稿。

## 3:30-4:30 实验结果

当前 active 模型训练池为 4691 张图片，验证集采用 source-holdout clean validation。

核心 clean validation 指标为 Accuracy 0.708、Macro-F1 0.503、GPT-image-2 Recall 0.915。

最新 full baseline 复测显示，active 在 clean/JPEG/截图/裁剪/水印 6 条件下的扰动平均 Macro-F1 为 0.655；截图转存下降最明显，GPT-image-2 recall 在 screenshot_resave 下为 0.222。候选模型 20ba33df 未自动激活，但严格门控结论为 suggest_activate。

进一步的技术线实验表明，低误报真实/生成初筛是更适合作品推进的方向。阈值校准候选在 360 条扰动验证上把真实图误报率压到 0.033，同时生成图召回为 0.692。GPT-image-2 专项组件 clean 表现很强，但来源互留仍未达标，因此我们只把它作为疑似线索，不作为确定归因。

同时我们也披露局限：strict source-holdout 均值 Macro-F1 为 0.124，label-covered 诊断 Macro-F1 为 0.354，说明公开数据源之间存在明显分布差异；后续要继续补 GPT-image-2 强扰动样本、真实照片 hard negatives 和外部盲测。

## 4:30-5:00 警务价值与边界

本项目的价值不是替代人工定案，而是把疑似生成来源线索转化为可复核证据链、处置建议和报告草稿。

系统不声称确定某图片来自 GPT-image-2，不替代 C2PA、水印、平台元数据、发布链路和人工取证。它适合作为公安机关开展网络谣言核查、平台协查、证据留存和风险研判的辅助工具。

# 答辩问答稿

## 1. 这是不是套壳项目？

不是。核心工作是本地视觉检测/归因头训练与鲁棒评估，包括外部数据集导入、特征抽取、扰动增强、feature cache、candidate/active 生命周期、source-holdout、阈值校准、feature-ablation 和模型卡审计。大模型报告只是应用层展示。

## 2. 是否训练了 Qwen3-VL？

没有训练基础多模态大模型本体。当前成果是本地监督视觉归因头和证据链系统，材料中只如实说明这一点。

## 3. 指标是不是作弊？

不是。增强样本只进入训练 split，验证集保持 clean holdout；内置四个展示方向样例不进入训练集。我们也披露了 source-holdout 泛化不足和 screenshot_resave 条件不稳定的问题。

## 4. GPT-image-2 结论能不能直接当证据？

不能。系统只输出疑似来源线索。最终仍需结合 C2PA、水印、平台元数据、发布账号链路、原始文件流转和人工核验。

## 5. 警务场景怎么落地？

落地方式是辅助研判：固定图像和 URL 证据，生成来源线索，组织证据链，给出处置建议，输出带 hash、模型版本和审计 ID 的报告草稿，供人工复核和平台协查使用。
